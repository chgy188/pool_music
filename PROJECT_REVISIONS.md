# Three.js Water Simulation - 项目修订说明

## 📋 项目概述

这是一个基于 **Three.js** 的 WebGL 水模拟项目，是 [Evan Wallace's webgl-water demo](http://madebyevan.com/webgl-water) 的 Three.js 实现版本。项目实现了逼真的水面渲染、波浪物理模拟和焦散(caustics)效果，并添加了音乐节奏响应功能。

### 核心技术栈
- **Three.js v0.113.0**: 3D 渲染引擎
- **WebGL**: GPU 加速图形渲染
- **GLSL**: 着色器语言，用于自定义渲染效果
- **GPGPU**: 通用 GPU 计算，用于水波物理模拟
- **Web Audio API**: 音频分析与节拍检测

## 🎵 音乐控制面板

### 功能特性

1. **导入本地音乐**
   - 点击 "📁 导入音乐" 按钮
   - 选择本地音频文件（支持 MP3、WAV、OGG 等格式）
   - 音乐会自动开始播放

2. **播放控制**
   - **播放/暂停**: 点击 "▶ 播放" 或 "⏸ 暂停" 按钮
   - **自动播放**: 导入音乐后自动开始播放
   - **播放完成**: 音乐结束后显示提示信息

3. **智能显示/隐藏**
   - **自动显示**: 鼠标移动时面板自动显示
   - **自动隐藏**: 鼠标静止 3 秒后面板自动隐藏
   - **悬停保持**: 鼠标悬停在面板上时保持显示
   - **平滑过渡**: 淡入淡出动画效果

### 效果强度控制

在音乐控制面板中可以调节音乐节拍驱动水波效果的强度：

- **0%**: 完全禁用音乐节拍水波
- **25%**: 非常微妙的涟漪，适合安静的音乐
- **50%**: 适中的水波，平衡的视觉效果
- **70%**: 默认强度，清晰可见的水波
- **100%**: 最大的水滴和最强的涟漪，适合强烈的电子音乐

### 节拍检测阈值

可以调节节拍检测的灵敏度：
- **较低值**: 更敏感（更多水波）
- **较高值**: 不太敏感（更少水波）

## 🌊 水波效果

### 鼠标交互
- **点击**: 在鼠标位置产生水波
- **拖动**: 连续产生水波轨迹
- **水波参数**: 半径 0.03，强度 0.04

### 音乐节拍水波
- **触发方式**: 自动检测音乐节拍（低频能量变化）
- **位置**: 随机分布在水面上
- **参数映射**:
  - 半径: 0.02 + intensity × 0.03
  - 强度: 0.03 + intensity × 0.05
  - 数量: floor(intensity × 2) + 1
- **节流机制**: 最小间隔 250ms，避免过度密集

### 轨迹效果
- **触发概率**: 30%（音乐节拍时）
- **轨迹长度**: 5-12 个水波（根据强度）
- **间隔**: 30ms
- **效果**: 类似鼠标拖拽的连续水波轨迹

## 🔧 技术架构

### 核心类

1. **WaterSimulation** - 水波物理模拟
   - 使用 GPGPU 技术模拟水波的物理行为
   - 使用 ping-pong 缓冲技术避免读写冲突
   - RGBA 纹理存储波形数据（高度、速度、法线）

2. **Caustics** - 焦散效果计算
   - 计算光线通过水面折射后在池底形成的焦散图案
   - 利用三角形面积变化率计算光线聚焦/发散程度

3. **Water** - 水面渲染
   - 渲染具有反射、折射效果的逼真水面
   - 使用菲涅尔效应混合反射和折射
   - 迭代采样使水面更"尖锐"

4. **Pool** - 泳池几何体
   - 渲染泳池的墙壁和底部
   - 使用瓷砖纹理贴图
   - 根据水深应用水下颜色调制

### 音频分析流程

```
音乐文件 → HTML5 Audio Element → Web Audio API
  ↓
MediaElementSourceNode → AnalyserNode (FFT: 256)
  ↓
频率数据分析（每帧 ~16ms）
  ├─ getByteFrequencyData()
  ├─ 分段计算平均值（低/中/高频）
  └─ 归一化到 [0, 1]
  ↓
节拍检测算法
  ├─ 计算能量变化
  ├─ 与阈值比较
  └─ 判断是否触发
  ↓
水滴参数生成
  ├─ 位置: 随机
  ├─ 半径: 根据强度
  ├─ 强度: 根据强度
  └─ 数量: 根据强度
  ↓
调用 addDrop() → 水面产生波纹
```

## 🐛 常见问题

### Q1: 音乐无法播放？
**A**: 检查以下几点：
1. 文件格式是否支持
2. 文件是否损坏
3. 浏览器是否允许自动播放
4. 查看控制台错误信息

### Q2: 面板不显示？
**A**:
1. 确认鼠标确实在移动
2. 检查浏览器控制台是否有错误
3. 尝试刷新页面

### Q3: 水池不随窗口调整？
**A**:
1. 打开浏览器控制台 (F12)
2. 查看是否有错误信息
3. 刷新页面 (Ctrl+R)

### Q4: 调整强度后水波无变化？
**A**:
- 音乐未播放
- 音频分析器未初始化
- 水波系统未就绪
